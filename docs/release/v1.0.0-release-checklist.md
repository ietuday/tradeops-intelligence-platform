# v1.0.0 Release Checklist

Use this checklist before tagging the GitHub release.

## Validation

- [ ] Surveillance service tests passed:

  ```bash
  (cd services/surveillance-service && go test ./...)
  ```

- [ ] Notification service tests passed:

  ```bash
  (cd services/notification-service && go test ./...)
  ```

- [ ] API Gateway tests passed:

  ```bash
  (cd services/api-gateway && npm test -- --runInBand)
  ```

- [ ] Docker Compose config passed:

  ```bash
  docker compose -f infrastructure/docker/docker-compose.yml config
  ```

- [ ] Demo scripts validated:

  ```bash
  bash -n scripts/demo-surveillance.sh
  bash -n scripts/demo-notifications.sh
  bash -n scripts/demo-e2e-tradeops.sh
  ```

- [ ] Smoke test script validated:

  ```bash
  bash -n scripts/smoke-test.sh
  ```

## Documentation

- [ ] README is GitHub-ready.
- [ ] Architecture docs are complete.
- [ ] API summary is complete.
- [ ] Event-flow docs are complete.
- [ ] Production-readiness checklist is complete.
- [ ] Troubleshooting guide is complete.
- [ ] Interview walkthrough and resume bullets are complete.
- [ ] Release notes exist for `v1.0.0` and `v1.0.1`.

## Repository Hygiene

- [ ] No secrets are committed.
- [ ] `infrastructure/docker/.env` is not committed.
- [ ] Generated files are removed or ignored.
- [ ] Local logs are removed.
- [ ] `node_modules/`, `dist/`, `.venv/`, `__pycache__/`, `.pytest_cache/`, `coverage/`, and `tmp/` are ignored.
- [ ] Required lock/dependency files remain tracked where applicable.

## Suggested Release Tag

```bash
git status --short
git add .
git commit -m "chore: prepare v1.0.0 portfolio release"
git tag -a v1.0.0 -m "v1.0.0 Production Readiness & Platform Hardening"
git push origin main
git push origin v1.0.0
```

Adjust branch name if the default branch is not `main`.
