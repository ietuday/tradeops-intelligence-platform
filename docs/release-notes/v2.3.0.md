# v2.3.0 - OpenTelemetry Distributed Tracing

## Added

- Jaeger all-in-one for local Docker Compose tracing.
- OpenTelemetry setup for API Gateway.
- OpenTelemetry setup for `order-service`, `surveillance-service`, `notification-service`, and `audit-service`.
- W3C `traceparent` / `tracestate` propagation through API Gateway proxy requests.
- Lightweight Kafka `traceparent` propagation for selected producers and consumers.
- Optional trace fields on emitted events where practical.
- `scripts/demo-otel-tracing.sh`.
- OpenTelemetry overview and runbook documentation.
- Optional Helm values for external OTLP exporter configuration.

## Preserved

- Existing `X-Correlation-ID` behavior remains intact.
- Existing business APIs, event handling, and audit lookup behavior are unchanged.

## Known Limitations

- Python services are future tracing candidates.
- Kafka propagation is payload/header based, not full automatic Kafka instrumentation.
- Audit persistence remains correlation-ID centric.
- No production-heavy tracing backend is added beyond local Jaeger.

No breaking changes.
