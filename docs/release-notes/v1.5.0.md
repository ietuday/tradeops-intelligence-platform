# v1.5.0 Data Retention, Archival & Event Replay

## Summary

v1.5.0 adds local operational data lifecycle support for TradeOps. The release focuses on documentation and safe scripts for retention planning, PostgreSQL backup/restore, archival exports, sample event replay, and DLQ replay guidance.

## Added

- Data lifecycle documentation under `docs/data-lifecycle/`:
  - Retention policy
  - Archival strategy
  - Backup and restore guide
  - Event replay guide
  - DLQ replay guide
  - Data lifecycle runbook
- PostgreSQL backup script: `scripts/db-backup.sh`.
- PostgreSQL restore script: `scripts/db-restore.sh`, guarded by `--confirm`.
- Old-data archival export script: `scripts/archive-old-data.sh`, dry-run/export by default and guarded by `--delete-confirm` for deletes.
- Sample event replay script: `scripts/replay-sample-events.sh`.
- Conservative DLQ replay helper: `scripts/replay-dlq-events.sh`.

## Updated

- README now includes data retention, backup, archive, sample replay, and DLQ replay commands.
- Smoke test now verifies lifecycle helper scripts exist without running destructive operations.
- Troubleshooting, production readiness, interview walkthrough, and DLQ docs now reference retention, backups, archival, and replay safety.

## Safety Notes

- No data is deleted automatically.
- Restore requires `--confirm`.
- Archive deletion requires `--delete-confirm`.
- DLQ replay remains manual/conservative and should only happen after the root cause is fixed.

## Breaking Changes

None.
