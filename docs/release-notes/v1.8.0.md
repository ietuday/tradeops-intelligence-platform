# v1.8.0 - Security Hardening & Threat Modeling

## Summary

v1.8.0 improves the platform security posture with practical API Gateway controls, threat modeling, RBAC documentation, secrets guidance, a read-only security validation script, and release documentation. No new business services were added and no core business behavior was changed.

## Added

- STRIDE-style threat model in `docs/security/threat-model.md`.
- Target RBAC matrix in `docs/security/rbac-matrix.md`.
- API security guide in `docs/security/api-security.md`.
- Secrets-management guidance in `docs/security/secrets-management.md`.
- Security checklist in `docs/security/security-checklist.md`.
- Read-only `scripts/security-check.sh` for risky file and security-control checks.
- `make security-check` target.
- Security workflow step that runs the TradeOps security check.

## API Gateway Hardening

- Helmet security headers remain enabled.
- `x-powered-by` remains disabled.
- CORS is configurable with `CORS_ORIGIN`.
- Request body size limit is configurable with `REQUEST_BODY_LIMIT` and returns `413` with `correlationId`.
- Lightweight in-memory rate limiting is configurable with `RATE_LIMIT_WINDOW_MS` and `RATE_LIMIT_MAX_REQUESTS` and returns `429` with `correlationId`.
- Tests cover security headers, CORS, request body limit, and rate limiting behavior.

## Documentation

- README security section and links.
- Production readiness checklist updates.
- Troubleshooting entries for rate limiting, request body size, CORS, and security check warnings.
- Interview walkthrough security hardening talking points.
- Kubernetes/Helm security configuration notes.
- Architecture security flow updates.

## Known Limitations

- Rate limiting is in-memory and local-demo oriented, not distributed production abuse protection.
- No OAuth/OIDC provider, mTLS, external secret manager, production WAF, or TLS ingress is added in this release.
- Local Docker Compose remains the default runtime.

## Compatibility

- No breaking changes.
- No frontend work.
- No Kubernetes/Helm feature expansion beyond documentation.
