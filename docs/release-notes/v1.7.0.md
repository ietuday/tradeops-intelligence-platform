# v1.7.0 Distributed Tracing & Correlation Visibility

## Summary

v1.7.0 adds lightweight distributed tracing readiness by standardizing correlation ID propagation across HTTP requests, gateway proxy calls, Kafka events, logs, DLQ records, audit logs, demo scripts, and documentation.

## Added

- Correlation ID standard documentation under `docs/tracing/`.
- Structured logging guidance for searching by `correlationId`.
- Tracing runbook for API, Kafka, DLQ, notification, and audit flows.
- Safe demo script: `scripts/demo-correlation-tracing.sh`.
- API Gateway tests for generated/preserved/forwarded correlation IDs.
- Audit normalization tests for `correlationId`, `correlation_id`, and missing correlation IDs.

## Updated

- Market data HTTP routes now include correlation ID middleware.
- Strategy and risk services now return `X-Correlation-ID` and include `correlationId` in produced events where applicable.
- Sample event payloads include `correlationId: demo-correlation-123`.
- Replay and demo scripts can use `CORRELATION_ID`.
- README, architecture, observability, audit, reliability, troubleshooting, and interview docs now explain correlation tracing.

## Notes

- No Jaeger, Tempo, OpenTelemetry Collector, Loki, or other heavy tracing stack was added.
- Correlation IDs provide practical traceability, not span-level distributed tracing.
- OpenTelemetry remains a future improvement.

## Breaking Changes

None.
