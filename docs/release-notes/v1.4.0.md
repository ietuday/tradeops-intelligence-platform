# v1.4.0 Advanced Observability & SLO Dashboards

## Summary

v1.4.0 improves demo and interview readiness for platform operations by adding advanced Grafana dashboards, Prometheus alert rules, an observability demo script, and SLO-focused documentation.

## Added

- Grafana dashboards for platform overview, API Gateway, event processing, surveillance/notifications, and audit/compliance.
- Prometheus alert rules for service availability, gateway latency/errors, event processing failures, DLQ events, notification delivery failures, and audit ingestion failures.
- Observability documentation:
  - Metrics catalog
  - Grafana dashboard guide
  - Prometheus alert guide
  - SLO guide
  - Observability runbook
- Read-only `scripts/demo-observability.sh` for demo navigation, Prometheus checks, and safe query examples.

## Updated

- Prometheus Compose configuration now mounts and loads alert rules.
- README, architecture, troubleshooting, interview, and production-readiness docs now reference SLO dashboards and alerting.

## Known Limitations

- Alert thresholds are local-demo defaults and should be tuned from real production traffic.
- Consumer lag metrics are not yet implemented; dashboards use retry, failure, and DLQ metrics as the current event-health indicators.
- Distributed tracing is still future work.

## Breaking Changes

None.
