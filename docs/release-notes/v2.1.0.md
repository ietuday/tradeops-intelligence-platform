# v2.1.0 - Real-Time WebSocket Streaming Layer

## Summary

v2.1.0 adds a lightweight real-time streaming layer inside the existing API Gateway so clients can receive live platform events without polling. REST API behavior remains unchanged.

## Added

- WebSocket endpoints:
  - `/ws`
  - `/ws/market`
  - `/ws/orders`
  - `/ws/alerts`
  - `/ws/notifications`
  - `/ws/audit`
- Kafka topic broadcasting for market ticks, order lifecycle events, surveillance alerts, notifications, and audit log events.
- JWT/RBAC checks for WebSocket upgrades using query token or `Authorization` header.
- Connection lifecycle logging and heartbeat messages.
- Prometheus metrics for active connections, total connections, messages sent/failed, auth failures, and consumed Kafka events.
- Safe demo script: `scripts/demo-websocket-streams.sh`.
- WebSocket example messages under `docs/examples/websocket/`.
- Real-time docs and runbook under `docs/realtime/`.

## Configuration

- `WS_ENABLED`
- `WS_REQUIRE_AUTH`
- `WS_KAFKA_BROKERS`
- `WS_KAFKA_GROUP_ID`
- `WS_ALLOWED_ORIGINS`
- `WS_MAX_CONNECTIONS`
- `WS_HEARTBEAT_INTERVAL_MS`

## Compatibility

- No breaking REST API changes.
- No new business service.
- WebSocket Kafka consumer startup failures are logged and do not crash the gateway.
