# v0.8.0 - Trade Surveillance & Alerting Service

## Summary

v0.8.0 adds a Go surveillance service for rule-based trade monitoring, alert storage, alert lifecycle APIs, Redpanda event integration, and Prometheus metrics.

## Included

- Added `services/surveillance-service` with chi HTTP routing, pgx PostgreSQL access, JWT/RBAC validation, Redpanda consumers/producers, and Prometheus metrics.
- Added `surveillance_alerts` and `surveillance_rule_executions` PostgreSQL tables.
- Added initial in-code rules for large orders, rapid order submission, high cancel rate, risk score breaches, and abnormal price movement.
- Added alert lifecycle endpoints for listing, detail, acknowledge, resolve, dismiss, and summary counts.
- Added API Gateway routes under `/api/surveillance` with explicit allowlisted proxy mapping.
- Added Docker Compose, `.env.example`, and Prometheus integration for the surveillance service.

## Event Topics

Consumed:

- `order.created`
- `order.filled`
- `order.cancelled`
- `portfolio.updated`
- `risk.score.updated`
- `market.ticks`
- `strategy.signal.generated`

Published:

- `surveillance.alert.created`
- `surveillance.alert.acknowledged`
- `surveillance.alert.resolved`
- `surveillance.alert.dismissed`

## Local URL

- Surveillance service direct URL: `http://localhost:8090`
- API Gateway surveillance base path: `http://localhost:8080/api/surveillance`

## Not Included

- External rule engine
- Distributed rule state
- Notification delivery
- Kubernetes deployment
