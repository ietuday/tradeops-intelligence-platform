# v1.9.0 - Performance Testing, Load Testing & Capacity Planning

## Summary

v1.9.0 adds safe local performance testing support, benchmark planning, capacity-planning guidance, performance runbooks, optional k6 scenarios, and documentation updates. No business behavior changed and no new service was added.

## Added

- Performance docs under `docs/performance/`:
  - `load-testing.md`
  - `benchmark-plan.md`
  - `capacity-planning.md`
  - `performance-runbook.md`
  - `performance-results-template.md`
- Optional k6 scenarios under `tests/load/k6/`:
  - `gateway-health.js`
  - `order-flow.js`
  - `surveillance-flow.js`
  - `notification-flow.js`
  - `audit-flow.js`
- `scripts/run-load-tests.sh` for opt-in k6 execution with low local defaults.
- `scripts/perf-smoke.sh` for lightweight curl timing checks.
- Makefile targets:
  - `perf-smoke`
  - `load-test-gateway`
  - `load-test-all`

## Documentation Updates

- README performance testing and capacity planning section.
- Observability docs now call out p95 latency, 5xx rate, event failures, retries, DLQ metrics, and notification delivery failures during load tests.
- Helm/deployment docs now mention resource requests/limits, scaling checks, and stateful dependency bottlenecks.

## CI

- CI validates shell syntax for the new scripts.
- CI does not run k6 load tests by default.

## Known Limitations

- Local k6 scenarios are safe demo baselines, not production capacity benchmarks.
- Results depend on local CPU, RAM, disk, Docker resource settings, and background processes.
- Kafka consumer lag metrics are not fully instrumented yet; retry, failure, duplicate, and DLQ metrics are the current proxy signals.

## Compatibility

- No breaking changes.
- No new business service.
- No cloud infrastructure requirement.
