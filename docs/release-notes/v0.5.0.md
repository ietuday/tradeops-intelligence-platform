# v0.5.0 - Portfolio Management

## Added

- Added Go `portfolio-service` for portfolio holdings, cash balances, snapshots, exposure, and P&L APIs.
- Added JWT validation and role-based access for portfolio endpoints.
- Added Docker Compose integration with host port `8087`.
- Added API Gateway routes under `/api/portfolio`.

## Portfolio Updates

- Consumes `order.filled` events from Redpanda.
- BUY events decrease cash, increase holdings, and recalculate average buy price.
- SELL events validate holdings, increase cash, decrease holdings, and record realized P&L.
- Duplicate order events are ignored using the `processed_order_events` table.
- Portfolios are created lazily with `PORTFOLIO_INITIAL_CASH`.

## Messaging

- Consumes `order.filled`.
- Publishes `portfolio.updated`.
- Publishes `portfolio.snapshot.created`.

## Observability

- Added Prometheus metrics for portfolio updates, failures, holdings count, cash balance, realized P&L, unrealized P&L, event processing duration, and Kafka publish errors.
- Added Prometheus scrape target for `portfolio-service`.

## Testing

- Added Go tests for portfolio role access helpers.
- Added API Gateway tests for portfolio health, holdings, and metrics proxying.
- Extended smoke tests to create a filled order and verify portfolio holdings update.

## Known Limitations

- Unrealized P&L is reported as zero until live mark prices are introduced.
- Portfolio valuation uses average buy price as the holdings value basis.
- Strategy service, risk engine, notification service, AI assistant, and Kubernetes remain out of scope.

## Next Release Plan

- Continue with the next scoped service release without implementing strategy, risk engine, notification service, AI assistant, or Kubernetes ahead of schedule.
