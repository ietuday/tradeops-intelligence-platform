# v0.9.0 - Notification Service

## Summary

v0.9.0 adds a Go notification service and integrates it into the local TradeOps platform.

## Included

- Added `services/notification-service` with notification persistence, preferences, JWT/RBAC, health/readiness, Prometheus metrics, and REST APIs.
- Added Redpanda consumers for surveillance alert lifecycle topics.
- Added Redpanda producers for notification lifecycle topics.
- Added IN_APP delivery, mock EMAIL delivery, and WEBHOOK delivery with timeout and bounded retry.
- Added API Gateway proxy routes under `/api/notifications`.
- Added Docker Compose service on port `8091`.
- Added Prometheus scrape target for `notification-service:8091`.
- Added smoke-test checks for notification health and readiness.
- Added `scripts/demo-notifications.sh`.
- Added sample notification demo payloads under `docs/examples/notifications`.

## Event Topics

Consumed:

- `surveillance.alert.created`
- `surveillance.alert.acknowledged`
- `surveillance.alert.resolved`
- `surveillance.alert.dismissed`

Published:

- `notification.created`
- `notification.sent`
- `notification.failed`
- `notification.read`
- `notification.retry_requested`

## Local URL

- Notification service direct URL: `http://localhost:8091`
- API Gateway notification base path: `http://localhost:8080/api/notifications`

## Known Limitations

- Email delivery is mock/log-only.
- Webhook delivery is intentionally simple and local-demo oriented.
- Notification preferences are user-scoped; no admin preference management API is included yet.
