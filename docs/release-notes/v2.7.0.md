# v2.7.0 - Admin Console APIs & Platform Operations

v2.7.0 adds a backend-only admin operations layer in the API Gateway. No frontend UI is included.

## Highlights

- Added `/api/admin` route group with JWT/RBAC protection.
- Added health summary aggregation across API Gateway and downstream services with short configurable timeouts.
- Added static service registry and event topic catalog endpoints.
- Added DLQ summary endpoint for known DLQ topics and replay/runbook guidance.
- Added audit, surveillance alert, notification, and rule configuration summary endpoints with degraded downstream behavior.
- Added safe platform configuration summary with secret masking.
- Added admin request and health check Prometheus metrics without tenant/user/trace/correlation labels.
- Added OpenTelemetry span attributes for admin endpoint, tenant, and correlation context.
- Added read-only demo script: `scripts/demo-admin-ops.sh`.
- Added admin operations documentation and release notes.

## Configuration

- `ADMIN_API_ENABLED=true`
- `ADMIN_HEALTH_TIMEOUT_MS=1500`
- `ADMIN_SERVICE_REGISTRY_MODE=static`

## Safety Notes

Admin APIs are read-only by default. The optional `POST /api/admin/cache/refresh` action is explicitly RBAC-protected for `trading_admin` and returns a safe `202` no-op message unless downstream refresh endpoints are configured in a future release.
