# v1.6.0 Deployment Readiness: Kubernetes / Helm Optional Layer

## Summary

v1.6.0 adds an optional Helm chart and Kubernetes deployment-readiness documentation while keeping Docker Compose as the primary local demo runtime.

## Added

- Optional Helm chart at `infrastructure/helm/tradeops-platform/`.
- Kubernetes templates for namespace, ConfigMap, example Secret, ServiceAccount, Deployments, Services, optional ingress, and release notes.
- Deployment and ClusterIP Service manifests for all application services.
- Liveness and readiness probes for `/health` and `/ready`.
- Resource requests and limits.
- Prometheus scrape annotations on service pods.
- Optional API Gateway ingress, disabled by default.
- Helm validation script: `scripts/validate-helm.sh`.
- Kubernetes/Helm documentation and deployment readiness checklist.

## Updated

- README now references the optional Helm chart and deployment docs.
- Makefile includes Helm validation targets.
- CI documentation and workflow include Helm validation guidance.
- Production readiness, troubleshooting, architecture, and interview docs mention Kubernetes deployment readiness.

## Notes

- Docker Compose is not replaced.
- No service business logic changed.
- No real secrets were added; the chart uses placeholder example secrets only.
- Stateful infrastructure is documented as an external/managed dependency for Kubernetes.

## Breaking Changes

None.
