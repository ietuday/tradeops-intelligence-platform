# v1.2.0 Reliability, Resilience & Failure Handling

## Added

- Added retry and exponential backoff configuration for event-consuming services.
- Added dead-letter publishing for portfolio, surveillance, and notification event consumers.
- Added DLQ payload metadata with original topic, original payload, error message, service name, failure time, correlation ID, and retry count.
- Added reliability metrics for retries, DLQ publishing, processing attempts, processing duration, duplicate skips, and API Gateway upstream failures.
- Added configurable API Gateway proxy timeout with `502` unavailable and `504` timeout responses.
- Added focused tests for DLQ retry behavior, notification duplicate handling, webhook timeout failure, and gateway proxy failure handling.
- Added reliability docs for graceful shutdown, resilience patterns, and dead-letter topics.
- Added an optional `scripts/demo-reliability.sh` helper for safe local reliability checks.

## Improved

- Portfolio idempotency is now observable through duplicate skip metrics.
- Surveillance validates malformed JSON source events and skips duplicate alerts for the same source/rule/entity.
- Notification processing returns malformed event errors to the consumer retry/DLQ path and skips duplicate notifications per source event/type/channel.
- API Gateway preserves correlation IDs in upstream timeout and unavailable responses.

## Known Limitations

- DLQ replay is manual and intentionally documented rather than automated.
- Webhook delivery remains intentionally simple and does not include a full circuit breaker.
- v1.2.0 does not add Kubernetes, Helm, or new business features.

## Breaking Changes

- None.
