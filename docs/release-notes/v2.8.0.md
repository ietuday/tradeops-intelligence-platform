# v2.8.0 Surveillance Rule Simulation & Dry-Run Engine

## Added

- Dry-run surveillance rule simulation APIs:
  - `POST /api/v1/surveillance/rules/{ruleName}/simulate`
  - `POST /api/v1/surveillance/rules/simulate`
- Internal `services/surveillance-service/internal/simulation` package for request models, demo event loading, in-memory config overlays, dry-run orchestration, and sample match collection.
- Rule engine evaluation mode constants for `live` and `dry_run`.
- Simulation lifecycle event schemas for:
  - `surveillance.rule_simulation.requested`
  - `surveillance.rule_simulation.completed`
  - `surveillance.rule_simulation.failed`
- Prometheus metrics for simulation requests, duration, matches, and failures.
- Safe demo script: `scripts/demo-rule-simulation.sh`.
- Documentation for rule simulation APIs, metrics, events, troubleshooting, and interview walkthroughs.

## Safety

- Simulation reads live tenant-effective configs but does not mutate `surveillance_rule_configs`.
- Simulation uses a throwaway rule engine and does not update the live rule config cache.
- Simulation does not create live alerts.
- Simulation does not publish `surveillance.alert.*` events.
- Simulation does not trigger notification flows.

## Known Limitations

- Historical data loading currently uses deterministic demo/historical-style source events, not a production event warehouse.
- Rule names remain fixed; user-authored custom rules are still future work.
- Simulation lifecycle events are available for observability/audit-style integrations but are not consumed by notification-service.

## No Breaking Changes

- Existing rule config APIs, alert creation behavior, Kafka alert events, and notification flows are unchanged.
