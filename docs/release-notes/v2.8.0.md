# v2.8.0 - Advanced Risk Analytics & Stress Testing

v2.8.0 extends `risk-engine-service` with backend-only advanced risk analytics APIs. No frontend UI or new service is added.

## Added

- Portfolio stress testing API with supplied positions and scenarios.
- Built-in scenario catalog and named scenario execution API.
- Concentration risk analysis by symbol, sector, and asset class.
- Drawdown trend analysis from supplied historical portfolio values.
- Volatility shock simulation with deterministic risk-score impact.
- Structured risk recommendations for concentration, drawdown, downside shock, and volatility exposure.
- Tenant and correlation propagation in risk analytics responses and events.
- Risk analytics Kafka event publication through the existing risk producer:
  - `risk.stress_test.completed`
  - `risk.scenario.completed`
  - `risk.concentration.analyzed`
  - `risk.drawdown.analyzed`
- JSON Schemas and sample payloads for the new risk analytics events.
- Prometheus metrics for stress tests, scenario runs, concentration analyses, drawdown analyses, recommendations, and analytics latency.
- Safe demo script: `scripts/demo-risk-analytics.sh`.
- Documentation and stress testing runbook under `docs/risk/`.

## Safety And Compatibility

- Existing risk score, volatility, drawdown, VaR, recommendation, and anomaly APIs remain backward-compatible.
- Calculations are deterministic and avoid heavy financial dependencies.
- GET portfolio-specific endpoints use deterministic demo data until direct live portfolio integration is added.
- Metrics avoid tenant, user, portfolio, trace, and correlation labels.

## Limitations

- Analytics use supplied/demo positions rather than a production market data warehouse.
- Stress formulas are explainable approximations, not calibrated institutional VaR/stress models.
- Historical drawdown requires supplied values for direct calculations.
