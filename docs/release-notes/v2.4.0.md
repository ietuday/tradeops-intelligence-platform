# v2.4.0 - Database Migration Runner & Seed Management

## Summary

v2.4.0 introduces a lightweight SQL-based migration runner and idempotent seed management for the shared local PostgreSQL platform database.

## Added

- `infrastructure/database/migrations` with versioned baseline schemas.
- `infrastructure/database/seeds` with demo-safe idempotent seed data.
- `scripts/db-migrate.sh` with `schema_migrations` tracking and checksum validation.
- `scripts/db-seed.sh` for repeat-safe demo seed execution.
- `scripts/demo-db-migrations.sh` to prove repeated migration/seed runs are safe.
- Optional Docker Compose `db-migrate` and `db-seed` tool services under the `tools` profile.
- Database migration, seed management, and troubleshooting docs.

## Changed

- README and docs index now link to the database setup workflow.
- CI/script validation includes the new database scripts.

## Validation

- `bash -n scripts/db-migrate.sh`
- `bash -n scripts/db-seed.sh`
- `bash -n scripts/demo-db-migrations.sh`
- `docker compose --env-file infrastructure/docker/.env.example -f infrastructure/docker/docker-compose.yml config`

## Known Limitations

- No rollback framework is included.
- Host execution requires `psql`; Compose tool services can be used when `psql` is not installed locally.
- Baseline infrastructure migrations coexist with service-owned migration folders.
