# v0.3.0 - Market Data Streaming

## Added

- Added Go `market-data-service` with MQTT tick ingestion, validation, normalization, PostgreSQL persistence, and Docker Compose integration.
- Added local market simulator for `AAPL`, `TSLA`, `MSFT`, `BTC-USD`, `ETH-USD`, `NIFTY50`, and `BANKNIFTY`.
- Added optional read endpoints for latest ticks and known symbols.
- Added API Gateway routes under `/api/market`.

## Messaging

- Subscribes to MQTT topic `market/+/tick`.
- Publishes normalized tick events to Redpanda Kafka topic `market.ticks`.

## Observability

- Added Prometheus metrics for tick receive, validation, publish, processing duration, MQTT connection status, and Kafka publish errors.
- Added Prometheus scrape target for `market-data-service`.

## Testing

- Added Go validation tests for market tick parsing and rejection rules.
- Added API Gateway proxy tests for market health, readiness, and metrics.
- Extended smoke tests for direct market health and API Gateway market health/readiness.

## Known Limitations

- OHLC table is created for persistence readiness, but candle aggregation is not enabled yet.
- Simulator is intentionally simple and only intended for local development.
- Market data auth and authorization are not enforced at the API Gateway in this release.

## Next Release Plan

- Continue with the next scoped service release without adding order, portfolio, strategy, risk, notification, AI assistant, or Kubernetes capabilities ahead of schedule.
