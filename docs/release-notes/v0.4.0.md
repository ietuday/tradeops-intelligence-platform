# v0.4.0 - Order Management

## Added

- Added Go `order-service` for simulated trading order management.
- Added JWT validation, RBAC enforcement, idempotency keys, PostgreSQL persistence, and Redpanda event publishing.
- Added API Gateway routes under `/api/orders`.
- Added Docker Compose integration with host port `8086`.

## Order Lifecycle

- MARKET orders are accepted and filled immediately with a simulated fill price.
- LIMIT and STOP_LOSS orders are accepted and remain open in this release.
- Invalid orders are persisted as rejected orders with a reject reason.
- Accepted orders can be cancelled; filled, rejected, and cancelled orders cannot be cancelled.

## Messaging

- Publishes order events to Redpanda topics:
  - `order.created`
  - `order.validated`
  - `order.accepted`
  - `order.filled`
  - `order.rejected`
  - `order.cancelled`

## Observability

- Added Prometheus metrics for order creation, acceptance, fills, rejections, cancellations, idempotency replays, Kafka publish errors, and processing duration.
- Added Prometheus scrape target for `order-service`.

## Testing

- Added Go tests for order validation and idempotency request hashing.
- Added API Gateway tests for order route forwarding and safe path handling.
- Extended smoke tests to authenticate, create a MARKET order, replay the idempotency key, and submit an invalid order.

## Known Limitations

- Order execution is simulated; no broker connectivity exists in v0.4.0.
- LIMIT and STOP_LOSS orders are accepted but not matched or triggered yet.
- Portfolio, strategy, risk, notification, AI assistant, and Kubernetes capabilities remain out of scope.

## Next Release Plan

- Continue with the next scoped service release without implementing portfolio, strategy, risk engine, notification service, AI assistant, or Kubernetes ahead of schedule.
