## Longer LinkedIn Post

TradeOps Intelligence Platform is my portfolio project for senior backend/system design interviews. It simulates a trading operations backend with order lifecycle events, portfolio updates, risk scoring, surveillance alerts, notifications, and audit logs.

The system uses a gateway plus domain services, PostgreSQL for durable data, Redis for identity state, MQTT for market ticks, and Redpanda/Kafka for event-driven workflows. I also added the operational pieces that make a project interview-ready: CI/CD validation, Prometheus/Grafana dashboards, SLO/runbook docs, retries/DLQ guidance, backup/replay scripts, security threat modeling, RBAC docs, Helm templates, and k6-based local load testing.

The goal was to show practical backend engineering judgment: clear boundaries, observable workflows, safe demos, and honest production limitations.
