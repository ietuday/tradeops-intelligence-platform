# v3.0.0 - Cloud-Native Production Deployment Blueprint

v3.0.0 introduces a cloud-neutral Kubernetes and Helm deployment blueprint for TradeOps.

Highlights:

- Helm chart versioned `3.0.0`
- local, staging, and production values profiles
- Kubernetes Deployments and Services for existing application components
- ConfigMaps, Secret references, optional development inline Secrets, and External Secrets example
- migration and seed Jobs
- Ingress, HPA, PDB, NetworkPolicy, ServiceMonitor, and optional OpenTelemetry Collector support
- Kind workflow scripts and Makefile targets
- production deployment, secrets, observability, network security, HA, migration, troubleshooting, and DR docs

Runtime validation fixes:

- local Kind deployment now builds and loads first-party `tradeops/*` images automatically
- backend Docker images use numeric non-root `USER 10001:10001`
- Helm security contexts set matching numeric UID/GID defaults for backend workloads
- frontend nginx workloads keep their image-native `101:101` security context
- local PostgreSQL, Redis, Redpanda, Mosquitto, migration, and seed workloads have restricted-compatible security contexts where image-compatible
- local Redpanda no longer uses the unsupported `--mode=dev-container` flag with `redpandadata/redpanda:v24.1.6`
- Kubernetes status and smoke scripts now fail earlier with clearer diagnostics for image pulls, container config errors, failed Jobs, and stuck rollouts

Upgrade sequence:

1. Back up PostgreSQL.
2. Prepare Kubernetes Secrets.
3. Configure external dependencies.
4. Render and review production manifests.
5. Run the migration Job.
6. Deploy stateless services.
7. Validate health, auth, RBAC, tenant behavior, dashboard, and WebSocket flow.
8. Enable ingress traffic and monitor errors, latency, saturation, and traces.

Known limitations:

- The chart is a production blueprint, not a managed cloud infrastructure stack.
- Production sizing, DNS, TLS, credentials, backups, and operational ownership remain environment-specific.
- Database rollback is not claimed because down migrations are not implemented and tested.
